     B. Permasalahan Penyimpanan Tag  
Sebelumnya tag disimpan dalam format JSON  di tabel posts.  
Contoh:  
tags 
---------  
["Laravel 12","PHP"]  
Kelemahan metode ini:  
• Data sulit dimodifikasi  
• Duplikasi data  
• Tidak efisien untuk query database  
• Tidak terstruktur  
Solusi:  
Menggunakan Many -to-Many Relationship  dengan tabel pivot .

     E. Membuat Tabel Tags  
Buka file create_posts_table.php dan tambahkan kode pada  migration:  
• Schema::create('tags', function (Blueprint $table) {  
    $table ->id();  
    $table ->string('name');  
    $table ->timestamps();  
}); 

=== PAGE 5 ===
 
 
     F. Membuat Pivot Table  
Buka file create_posts_table.php dan tambahkan kode pada  migration:  
• Schema::create('post_tag', function (Blueprint $table) {  
    $table ->foreignId('post_id')  
        ->constrained()  
        ->cascadeOnDelete();  
    $table ->foreignId('tag_id')  
        ->constrained()  
        ->cascadeOnDelete();  
    $table ->primary(['post_id','tag_id']);  
});

     K. Menggunakan Relationship pada Form Post

     M. Membuat Relationship Manager  
Relationship Manager memungkinkan mengelola tag langsung dari post.  
Jalankan perintah:  
• php artisan make:filament -relation -manager  
Isi parameter:

Relationship : tags  
Column       : name
